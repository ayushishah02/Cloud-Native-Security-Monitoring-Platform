# Phase 4 — Security Monitoring & Observability

## Overview

Phase 4 implements centralized logging, security monitoring, and detection capabilities for the **Cloud-Native Security Monitoring Platform**.

The goal of this phase is to simulate authentication traffic, collect application logs from Kubernetes workloads, store them in a centralized logging system, visualize security events, and create alerting rules to detect suspicious activity.

This phase demonstrates how modern **cloud-native security monitoring pipelines** work in real-world DevSecOps and SOC environments.

---

# Architecture

Traffic Generator (Kubernetes Job) -> SecureWorks Web Application -> Application JSON Logs -> Grafana Alloy (Log Collector) -> Loki (Centralized Log Storage) -> Grafana (Visualization + Alerting)

---

# Key Components

## 1. Traffic Generator

A Kubernetes job simulates authentication activity against the application.

The generator produces:

- Normal login attempts
- Failed login attempts
- Brute-force style login patterns

Location:
phase4/k8s/jobs/traffic-gen-brute.yaml

The job runs temporarily, generates traffic, and then exits.

Example log event generated by the application:

```json
{
  "ts": "2026-03-16T02:45:06.895708+00:00",
  "event": "auth_attempt",
  "result": "success",
  "reason": "ok",
  "username": "alice",
  "src_ip": "198.51.100.10",
  "user_agent": "Mozilla/5.0",
  "path": "/api/login",
  "scenario": "normal",
  "service": "secureworks",
  "env": "docker-desktop"
}
```
These structured logs enable downstream parsing and analysis.

---
## 2. Log Collection (Grafana Alloy)
Grafana Alloy collects container logs from the Kubernetes cluster and forwards them to Loki.

Configuration file: phase4/k8s/logging/alloy-values.yaml

# Responsibilities:

- Discover Kubernetes pods
- Collect container logs
- Label log streams
- Forward logs to Loki

Alloy acts as the log ingestion layer of the monitoring stack.

--- 
## 3. Log Storage (Loki)
Loki provides centralized log storage designed for Kubernetes workloads.

Configuration File: phase4/k8s/logging/loki-values.yaml

Loki stores logs with labels such as:

- namespace
- pod
- container
- job

This enables efficient querying using LogQL.

Unlike traditional log platforms, Loki indexes metadata rather than full log contents, making it lightweight and scalable.

---
## 4. Visualization (Grafana)

Grafana dashboards visualize authentication activity across the system. 

Grafana dashboards visualize authentication activity across the system.

The dashboards provide visibility into:

- Failed login attempts over time
- Successful login attempts
- Top attacking IP addresses
- Most targeted usernames

These panels allow security teams to quickly identify abnormal authentication behavior.
